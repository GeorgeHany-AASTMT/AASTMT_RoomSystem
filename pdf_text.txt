 ○  Has  the  privilege  to  view  all  available  and  booked  rooms  and  time  slots.  ○  Can  directly  modify  any  lecture  room  booking  without  requiring  third-party  approval.  ○  Receives  booking  requests  and  possesses  the  authority  to  approve  or  reject  them

7.  User  Interface  (UI)  &  Navigation  
To  ensure  a  seamless  user  experience,  the  interfaces  should  be  designed  as  follows:  
A.  Standard  Users  Interface  (Employee  /  Secretary):  
●  Simplified  Dashboard:  Displays  quick  statistics  and  a  history  of  their  requests  (Pending,

 ●  New  Request  Form:  A  clean  interface  to  fill  out  the  booking  details  (as  outlined  in  section

 ●  Hidden  Availability:  No  calendar  grids  showing  occupied  or  available  rooms  are  visible  by  
default.
 
B.  Admin  Dashboard  (Room  Manager):  
The  Admin  dashboard  is  the  core  of  the  system  and  must  include:  ●  Calendar  Grid:  ○  Weekly  View:  Displays  days  of  the  week  as  columns  and  time  slots  as  rows.  Booked

 ●  Advanced  Search  &  Filters  (For  Empty  Rooms):  ○  A  dedicated  interface  for  the  Admin  to  search  specifically  for  availability.  ○  Filters:  Select  (Specific  Date)  +  (Specific  Time  Slot)  +  (Room  Type).  ○  Result:  Returns  a  list  of  room  names  that  have  zero  confirmed  bookings  during  that
